Instructions for running the LoadInvestorIndustries Console App

Requirements and Dependencies

  • .NET Framework 4.5
  • Enterprise Library Logging Component Version 5.0.505.0
  • Windows Azure SDK 2.1
  • Json.NET 4.5
  • SQL Server or Azure SQL database

Build and Configuration

  1. Create the database objects by running "SQL Scripts\LoadInvestorIndustries_Create.sql" in the ScreenVc database. This will create three tables, three user-define table types, and three stored procedures.

  2. Open the Visual Studio solution and edit the configuration file connection string Screen.Vc.DataAccess.Investors.Properties.Settings.ScreenVcConnectionString. The "reference" config section is in Screen.Vc.DataAccess.Investors. The config section used by LoadInvestorIndustries.exe is in LoadInvestorIndustries\App.config.

Be sure to set MultipleActiveResultSets=True. You may want to set a minimum connection pool size.

  1. By default, two log files are written to the working directory of the application. If you want to change this, edit the Enterprise Logging configuration. Two logs are written: LoadInvestorIndustriesGeneral.log for verbose log messages, and LoadInvestorIndustriesWarning.log for severity of Warning or higher.

  2. Using Visual Studio 2012, build the project. Dependencies should be present in the solution, but can be installed via NuGet.

  3. Run "LoadInvestorIndustries.exe -?" for usage. Parameters "155 160" give a good sample of investor and non-investor ids. (For debugging, set LoadInvestorIndustries to be the startup project and in Properties/Debug set command line arguments to something reasonable like "155 160".)

Additional Configuration

  • In LoadInvestorIndustries.Properties.Settings, use MaxRangeLength and MaxRanges to control the size and number of batches of user ids fetched.

Running the Unit Tests in LoadInvestorIndustries.Test.

In order to verify the data saved to the database, the unit tests delete all data from the tables. Only run the test against a test database.
